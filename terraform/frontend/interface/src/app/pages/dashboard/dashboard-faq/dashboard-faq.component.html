Frequently Asked Questions

  1. How can I get access to the SDC Datasets?

    Users can request access to the datasets that are available within the SDC platform as published / enabled by the SDC team or published by other users.

    1. Once you are logged in, click on DATASETS in the top menu.
    2. Expand the SDC Datasets panel. Below this you can see all available datasets.
    3. To access a given Dataset, click on Request.
    4. The SDC Data Access Request Form will pop-up. Complete the form and click Send Request.
    5. The request will be sent to the support team and access to the requested dataset will be given upon validataion and approval of the information in the form.
  2. How can I understand what a particular dataset consists of?
    Click anywhere on a given Dataset entry for the corresponding README description to display below it.
  3. How do I launch a workstation?

    Users can see their assigned workstations by clicking on WORKSTATIONS from the top menu. By default, all workstations are in a stopped state.

    1. Click on the Start button to start your workstation machine.
    2. The workstation should become available within a few minutes, confirmed by the following message.

    3. Click the corresponding Launch button for your workstation.

      Note: The Launch button does not start your workstation. You must start it before you can log in.

    4. This opens the workstation login page. Enter your SDC username and password at the prompts to access your workstation desktop.

      Note: The workstation may take a few minutes to initialize before the login screen appears.

  4. How do I upload my own datasets/algorithm?

    Users who want to share data with other users from their project team can upload their own data to their assigned team/individual S3 buckets.

    1. Click on DATASETS on the top menu.

    2. Click on Upload Files under My Datasets/Algorithm.

    3. A pop-up window appears prompting you to choose one or more files for upload to the assigned bucket. (The assigned bucket name will be displayed on the upload pop-up window.)
    4. Once you have chosen your file(s), click the Upload button. A success message will be displayed upon successful upload.

    5. Files that are uploaded from the web portal will be saved in the folder username/uploaded_files.

    6. To make these files available to members of the project team, users then need to move files from username/uploaded files to the project team S3 bucket using the CyberDuck tool.

    7. Users would be able to access only the files that are under the uploaded_files and export_requests folders.

  5. How do I publish my dataset/algorithm?

    Follow the below steps to publish your datasets / algorithms and share with other SDC Users.

    1. Navigate to Datasets page and scroll to the My Datasets / Algorithms section

    2. Click on the upload icon under publish for the dataset/algorithm you wish to publish

    3. In the pop-up window that shows up, you will have two options either a Dataset or Algorithm:

      1. Select the dropdown value as Dataset and fill the required values

        1. Name - Name of the dataset, which you wish to call it. Users will see your dataset with this name under SDC Datasets section
        2. Description - Provide a short description so users can get an idea about your dataset
        3. File/folder name - Name of the file or folder where your dataset resides in your Home folder. We need this information, so the support team can publish this dataset and make it available to other users
        4. Readme / Data dictionary file name - This file should provide detailed instructions about your dataset, how it was created or any relevant information that helps user to understand and use the dataset. Save this file in your home folder relative to the dataset file/folder name
        5. Geographic scope - Indicate the geographic scope for your dataset whether it belongs to a specific state, region, country etc
        6. Start/End Date for data availability - Provide the start and end dates of the data that belongs in your dataset. For example, your dataset may contain data from March 2017 to August 2017

      2. Select the dropdown value as Algorithm

        1. Name - Enter the name for your algorithm. Users will see your algorithm with this name under SDC Datasets section
        2. Description - Provide a short description about your algorithm
        3. File/Folder name - Name of the file or folder where your algorithm resides in your Home folder. We need this information, so SDC support team can publish this algorithm and make it available to other users
        4. Readme / Data dictionary file name - This file should provide detailed instructions about your algorithm, how it was created or any relevant information that helps user to understand and use the algorithm. Save this file in your home folder relative to the algorithm file/folder name
        5. Programming Tools/language - Provide the details of programming tools and/or languages that were used to create this algorithm, so users can leverage the same to run your program.
